Memmem is probably faster than strstr, right?

ChatGPT

In general, memmem is often more efficient than strstr for certain use cases because it is designed for searching arbitrary memory regions, whereas strstr is specifically tailored for null-terminated strings.

memmem takes two memory regions and their lengths as parameters, allowing it to work with binary data or non-null-terminated strings. This flexibility makes it suitable for scenarios where you want to find occurrences of one block of memory within another.

On the other hand, strstr assumes null-terminated strings, so it looks for the occurrence of a substring within another string until it encounters a null character. If you are working with null-terminated strings and want to find one string within another, strstr is appropriate.

In summary, if you are dealing with binary data or non-null-terminated strings, memmem is likely to be more efficient. If you are working with null-terminated strings, strstr is the appropriate choice. Always consider the specific requirements and characteristics of your data when choosing between these functions.
